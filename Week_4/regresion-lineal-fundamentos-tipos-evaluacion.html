Regresión Lineal

Aprende a modelar relaciones entre variables para hacer predicciones. Desde la regresión simple con una variable hasta modelos múltiples con todo un conjunto de predictores, aplicados al dataset de Calidad del Agua.

1. Fundamentos de la Regresión Lineal

Regresión Lineal: Técnica de modelado estadístico que busca la relación lineal entre una variable dependiente (Y) y una o más variables independientes (X). El objetivo es encontrar la recta (o hiperplano) que mejor se ajusta a los datos observados, minimizando la suma de los errores al cuadrado.

Ecuaciones del modelo

Regresión Simple

Ŷ = β₀ + β₁X

Una sola variable predictora. La salida es una recta en el plano XY.

  • β₀ (intercepto): valor de Y cuando X = 0
  • β₁ (pendiente): cambio en Y por cada unidad de X

Regresión Múltiple

Ŷ = β₀ + β₁X₁ + β₂X₂ + ... + βₙXₙ

Múltiples variables predictoras. La salida es un hiperplano en el espacio multidimensional.

  • β₀: intercepto general
  • βᵢ: efecto de Xᵢ manteniendo las demás constantes

Supuestos de la Regresión Lineal

Para que los resultados sean válidos e interpretables, deben cumplirse 5 supuestos clave:

Supuesto Descripción Prueba
Linealidad Relación lineal entre X e Y Gráfico de dispersión
Independencia Los errores son independientes entre sí Durbin-Watson
Homoscedasticidad Varianza constante de los errores Breusch-Pagan
Normalidad Los residuos siguen distribución normal Shapiro-Wilk, Q-Q plot
No multicolinealidad Variables predictoras no altamente correlacionadas VIF (Factor de Inflación de Varianza)

Función de Costo: MSE

El modelo de regresión busca minimizar el Error Cuadrático Medio (MSE):

MSE = (1/n) × Σ(Yᵢ − Ŷᵢ)²

Es decir, busca la recta que hace los residuos (diferencias entre el valor real y el predicho) lo más pequeños posible en promedio.

💡 ¿Cuándo usar regresión lineal?

  • La variable objetivo es continua (numérica)
  • Se sospecha una relación lineal entre las variables
  • Se necesita un modelo interpretable (los coeficientes tienen significado directo)
  • Ejemplo: predecir la conductividad del agua a partir de sólidos disueltos

2. Regresión Lineal Simple

Modelamos la relación entre una sola variable predictora (X) y la variable objetivo (Y).

🔬 Pregunta de investigación

¿Podemos predecir la Conductividad del agua a partir de los Sólidos disueltos?

Estas dos variables tienen sentido físico: más sólidos disueltos (iones) generalmente implican mayor conductividad eléctrica.

Paso 1: Preparar los datos

📊 División entrenamiento/prueba

import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression

url = "https://raw.githubusercontent.com/jaquimbayoc7/material-fundamentos_datos/main/data/water_potability.csv"
df = pd.read_csv(url)

# Imputar nulos con mediana
for col in df.columns:
    if df[col].isnull().sum() > 0:
        df[col].fillna(df[col].median(), inplace=True)

# Definir variables
X_simple = df[['Solids']].values
y_simple = df['Conductivity'].values

# Dividir 80/20
X_train, X_test, y_train, y_test = train_test_split(
    X_simple, y_simple, test_size=0.2, random_state=42
)

print(f"Entrenamiento: {X_train.shape[0]} muestras")
print(f"Prueba:        {X_test.shape[0]} muestras")

Paso 2: Entrenar el modelo

📈 Ajuste y coeficientes

# Entrenar
modelo_simple = LinearRegression()
modelo_simple.fit(X_train, y_train)

# Resultados
print(f"Intercepto (β₀): {modelo_simple.intercept_:.4f}")
print(f"Pendiente  (β₁): {modelo_simple.coef_[0]:.6f}")
print(f"Ecuación: Conductivity = {modelo_simple.intercept_:.4f} + {modelo_simple.coef_[0]:.6f} × Solids")

# Predicciones
y_pred_train = modelo_simple.predict(X_train)
y_pred_test = modelo_simple.predict(X_test)

Paso 3: Visualizar la regresión

🎨 Datos + Recta de regresión

import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(16, 6))

axes[1].scatter(y_test, y_pred_test, alpha=0.3, s=15, color='coral')

axes[1].set_xlabel('Valores Reales')

Interpretación del modelo

📝 ¿Cómo leer los coeficientes?

  • Intercepto (β₀): Conductividad estimada cuando los sólidos disueltos son 0. Es el punto donde la recta cruza el eje Y.
  • Pendiente (β₁): Por cada unidad (ppm) de aumento en sólidos, la conductividad cambia en β₁ μS/cm.
  • Si β₁ es positivo → relación directa. Si es negativo → relación inversa.

3. Métricas de Evaluación

Las métricas nos indican qué tan bien se ajusta el modelo a los datos. Siempre debemos evaluarlas en el conjunto de prueba para estimar el rendimiento real.

Métrica Fórmula Interpretación
R² R² = 1 − (SS_res / SS_tot) % de varianza explicada (0 a 1, más alto = mejor)
MSE (1/n) × Σ(yᵢ − ŷᵢ)² Error promedio al cuadrado (penaliza errores grandes)
RMSE √MSE Error en las mismas unidades que Y
MAE (1/n) × Σ|yᵢ − ŷᵢ| Error promedio absoluto (más robusto a outliers)

R² (Coeficiente de Determinación)

Indica qué proporción de la variabilidad de Y es explicada por el modelo.

  • R² = 1.0: modelo perfecto
  • R² = 0.0: el modelo no explica nada
  • R² < 0: el modelo es peor que predecir la media

R² Ajustado

Penaliza por la cantidad de variables. Importante en regresión múltiple: evita que agregar variables inútiles infle el R².

R²_adj = 1 − [(1−R²)(n−1) / (n−p−1)]

Donde n = muestras, p = número de predictores.

RMSE vs MAE

RMSE penaliza más los errores grandes (sensible a outliers). MAE trata todos los errores por igual (más robusto).

Si RMSE >> MAE → hay errores grandes puntuales (outliers en predicción).

Código: Calcular métricas

📊 Evaluación del modelo simple

from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

# Métricas en entrenamiento y prueba
r2_train = r2_score(y_train, y_pred_train)
r2_test = r2_score(y_test, y_pred_test)
mse_test = mean_squared_error(y_test, y_pred_test)
rmse_test = np.sqrt(mse_test)
mae_test = mean_absolute_error(y_test, y_pred_test)

metricas = pd.DataFrame({
    'Métrica': ['R²', 'MSE', 'RMSE', 'MAE'],
    'Entrenamiento': [r2_train, mean_squared_error(y_train, y_pred_train),
                      np.sqrt(mean_squared_error(y_train, y_pred_train)),
                      mean_absolute_error(y_train, y_pred_train)],
    'Prueba': [r2_test, mse_test, rmse_test, mae_test]
})
display(metricas.round(4))

print(f"El modelo explica el {r2_test*100:.2f}% de la varianza")
print(f"Error promedio (RMSE): {rmse_test:.2f} μS/cm")

Gráfica Interactiva: Comparación de Métricas

Métricas del modelo simple (Solids → Conductivity) en entrenamiento vs prueba:

⚠️ ¿Cuándo preocuparse?

  • R² muy bajo (cercano a 0): las variables elegidas no explican Y → buscar otras variables o relaciones no lineales
  • R² train >> R² test: sobreajuste (overfitting) → el modelo memoriza los datos de entrenamiento pero no generaliza
  • RMSE muy alto vs el rango de Y: el modelo tiene errores grandes → considerar transformaciones o más variables

4. Regresión Lineal Múltiple

Incorporamos múltiples variables predictoras para mejorar la capacidad predictiva del modelo.

Selección de variables

Antes de construir el modelo, debemos verificar que las variables predictoras no estén altamente correlacionadas entre sí (multicolinealidad). Usamos el VIF:

📐 Verificación de multicolinealidad

import statsmodels.api as sm
from statsmodels.stats.outliers_influence import variance_inflation_factor

target = 'Conductivity'
features = [c for c in df.select_dtypes(include=[np.number]).columns
            if c != target and c != 'Potability']

X_vif = df[features].copy()
X_vif_const = sm.add_constant(X_vif)

vif_data = pd.DataFrame()
vif_data['Variable'] = features
vif_data['VIF'] = [variance_inflation_factor(X_vif_const.values, i+1)
                   for i in range(len(features))]
vif_data = vif_data.sort_values('VIF', ascending=False)
display(vif_data)

# Seleccionar features con VIF < 10
features_ok = vif_data[vif_data['VIF'] < 10]['Variable'].tolist()
print(f"Features seleccionadas: {features_ok}")

Entrenar el modelo múltiple

📈 Regresión con múltiples variables

# Preparar datos
X_multi = df[features_ok].values
y_multi = df[target].values

X_train_m, X_test_m, y_train_m, y_test_m = train_test_split(
    X_multi, y_multi, test_size=0.2, random_state=42
)

# Entrenar
modelo_multiple = LinearRegression()
modelo_multiple.fit(X_train_m, y_train_m)

# Coeficientes
print(f"Intercepto (β₀): {modelo_multiple.intercept_:.4f}")
for feat, coef in zip(features_ok, modelo_multiple.coef_):
    print(f"  {feat:25s} → β = {coef:+.6f}")

# Predicciones
y_pred_train_m = modelo_multiple.predict(X_train_m)
y_pred_test_m = modelo_multiple.predict(X_test_m)

Evaluación del modelo múltiple

📊 Métricas con R² ajustado

r2_test_m = r2_score(y_test_m, y_pred_test_m)
rmse_test_m = np.sqrt(mean_squared_error(y_test_m, y_pred_test_m))
mae_test_m = mean_absolute_error(y_test_m, y_pred_test_m)

# R² ajustado
n = X_test_m.shape[0]
p = X_test_m.shape[1]
r2_adj = 1 - (1 - r2_test_m) * (n - 1) / (n - p - 1)

print(f"R²:          {r2_test_m:.4f}")
print(f"R² ajustado: {r2_adj:.4f}")
print(f"RMSE:        {rmse_test_m:.4f}")
print(f"MAE:         {mae_test_m:.4f}")

5. Comparación: Simple vs Múltiple

Comparar modelos es fundamental para elegir el que mejor equilibra capacidad predictiva y complejidad.

⚖️ Tabla comparativa

comparacion = pd.DataFrame({
    'Modelo': ['Regresión Simple (Solids)', f'Regresión Múltiple ({len(features_ok)} vars)'],
    'R² (Test)': [r2_test, r2_test_m],
    'RMSE (Test)': [rmse_test, rmse_test_m],
    'MAE (Test)': [mae_test, mae_test_m]
})
display(comparacion.round(4))

mejora_r2 = (r2_test_m - r2_test) / abs(r2_test) * 100
print(f"Mejora en R²: {mejora_r2:+.2f}%")

Gráfica Interactiva: Simple vs Múltiple

Comparación visual de las métricas de ambos modelos:

✅ Ventajas del modelo simple

  • Fácil de interpretar
  • Requiere solo una variable
  • Menor riesgo de sobreajuste
  • Útil como punto de partida (baseline)

✅ Ventajas del modelo múltiple

  • Captura más variabilidad
  • Considera interacciones entre variables
  • Generalmente mejor R²
  • Más realista para fenómenos complejos

⚠️ Precauciones

  • Agregar más variables no siempre mejora el modelo — puede causar sobreajuste
  • Usar R² ajustado en vez de R² para comparaciones justas
  • Verificar que las variables extra sean significativas (p < 0.05)
  • Un R² bajo no significa un modelo inútil — puede indicar que las variables disponibles no capturan toda la variabilidad del fenómeno

7. Resumen y Puntos Clave

Regresión Simple

Una variable predictora. Ideal para explorar relaciones individuales y como baseline de comparación.

Regresión Múltiple

Múltiples predictores. Captura más complejidad pero requiere verificar multicolinealidad (VIF).

Métricas clave

R², R² ajustado, RMSE y MAE. Evaluar siempre en el conjunto de prueba para estimar rendimiento real.

Buenas prácticas

Dividir datos 80/20, verificar supuestos, comparar modelos y no agregar variables sin justificación.

🎯 Recuerda

La regresión lineal es una herramienta poderosa e interpretable, pero sus resultados solo son confiables si se validan los supuestos. En la siguiente sesión aprenderemos a validar cada supuesto y construir un pipeline profesional completo.
