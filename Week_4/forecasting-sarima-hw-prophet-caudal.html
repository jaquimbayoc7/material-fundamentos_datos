🔮 Semana 4: Forecasting — SARIMA, Holt-Winters y Prophet

Aplicamos tres modelos de pronóstico a la serie de caudal: SARIMA captura la estructura autorregresiva, Holt-Winters modela tendencia y estacionalidad con suavizado exponencial, y Prophet detecta patrones automáticamente. Comparamos métricas y seleccionamos el mejor modelo para proyectar los próximos 30 días.

1. Estrategia de Evaluación: Train / Test Split

Train/Test Split temporal: En series de tiempo, nunca se mezclan datos aleatoriamente. Se reservan los últimos n días como conjunto de prueba y se entrena con todo lo anterior. Esto simula un escenario real donde se predice el futuro sin conocerlo. 
# Últimos 60 días como test
train = df["Caudal"][:-60]
test  = df["Caudal"][-60:]
print(f"Train: {len(train)} días  |  Test: {len(test)} días")

🏋️ Entrenamiento (Train)

~2,862 días (2010 – finales de 2017). Se usa para ajustar los parámetros de cada modelo.

🧪 Prueba (Test)

Últimos 60 días de la serie. Se usan para evaluar la capacidad predictiva de cada modelo sobre datos no vistos.

💡 ¿Por qué 60 días?
Es un horizonte suficiente para evaluar si el modelo captura la dinámica estacional (2 meses abarcan diferentes comportamientos). Es un balance entre tener suficientes datos de test para una evaluación robusta y no perder demasiados datos de entrenamiento.

2. Modelo SARIMA

SARIMA (Seasonal AutoRegressive Integrated Moving Average) extiende ARIMA añadiendo componentes estacionales. Se define con 7 parámetros: SARIMA(p, d, q)(P, D, Q, m).

2.1 Componentes del modelo

ParámetroComponenteSignificado
pAR (Autorregresivo)Número de lags propios. x(t) depende de x(t−1), ..., x(t−p)
dI (Integración)Orden de diferenciación para lograr estacionariedad
qMA (Media Móvil)Número de lags del error. x(t) depende de ε(t−1), ..., ε(t−q)
PAR EstacionalLags autorregresivos en el período estacional
DI EstacionalDiferenciación estacional
QMA EstacionalLags de media móvil estacional
mPeríodoLongitud del ciclo estacional (30 para mensual)

2.2 Fórmula general

Φ(B^m) · φ(B) · (1−B)^d · (1−B^m)^D · x(t) = Θ(B^m) · θ(B) · ε(t)

Donde B es el operador de retardo (B·x(t) = x(t−1)), φ y θ son los polinomios AR y MA ordinarios, y Φ y Θ los estacionales.

2.3 Configuración aplicada

from statsmodels.tsa.statespace.sarimax import SARIMAX

model = SARIMAX(train,
                order=(1, 1, 1),
                seasonal_order=(1, 1, 1, 30),
                enforce_stationarity=False,
                enforce_invertibility=False)
fit = model.fit(disp=False)
pred_sarima = fit.forecast(steps=60)
📊 Interpretación de los parámetros:
  • (1,1,1): AR(1) + 1 diferenciación + MA(1) — el mínimo necesario según ACF/PACF
  • (1,1,1,30): Componente estacional con período de 30 días (ciclo mensual), ya que el período de 365 es computacionalmente costoso

3. Modelo Holt-Winters (Suavizado Exponencial Triple)

Holt-Winters: Método de suavizado exponencial que modela tres componentes con pesos exponencialmente decrecientes: nivel (α), tendencia (β) y estacionalidad (γ). No requiere estacionariedad previa.

3.1 Variantes

✅ Aditiva (seleccionada)

ŷ(t) = l(t) + b(t) + s(t−m)

La estacionalidad se suma al nivel. Apropiada cuando la amplitud estacional es constante en el tiempo.

Multiplicativa

ŷ(t) = l(t) · b(t) · s(t−m)

La estacionalidad se multiplica. Apropiada cuando la amplitud crece con el nivel de la serie.

3.2 Ecuaciones de actualización

Nivel (α)
l(t) = α·(y(t)−s(t−m)) + (1−α)·(l(t−1)+b(t−1))
Tendencia (β)
b(t) = β·(l(t)−l(t−1)) + (1−β)·b(t−1)
Estacionalidad (γ)
s(t) = γ·(y(t)−l(t)) + (1−γ)·s(t−m)

3.3 Aplicación

from statsmodels.tsa.holtwinters import ExponentialSmoothing

model_hw = ExponentialSmoothing(
    train,
    trend="add",
    seasonal="add",
    seasonal_periods=30
)
fit_hw = model_hw.fit()
pred_hw = fit_hw.forecast(steps=60)

4. Modelo Prophet (Meta/Facebook)

Prophet: Modelo desarrollado por Meta que descompone la serie en tendencia, estacionalidad y efectos de días festivos usando un modelo aditivo generalizado (GAM). Es robusto ante datos faltantes, cambios de tendencia y outliers.

4.1 Modelo matemático

y(t) = g(t) + s(t) + h(t) + ε(t)
ComponenteDescripción
g(t)Tendencia: crecimiento lineal o logístico con changepoints automáticos
s(t)Estacionalidad: series de Fourier para capturar patrones periódicos
h(t)Efectos de días festivos/eventos especiales
ε(t)Ruido/error irreducible

4.2 Ventajas de Prophet

🔧 Automático

Detecta changepoints y estacionalidad sin parametrización manual

📅 Multi-estacionalidad

Puede capturar estacionalidad diaria, semanal, anual simultáneamente

🛡️ Robusto

Tolerante a datos faltantes y outliers

📊 Intervalos

Proporciona intervalos de confianza (incertidumbre) automáticamente

4.3 Aplicación

from prophet import Prophet

# Prophet requiere columnas 'ds' (fecha) y 'y' (valor)
df_prophet = train.reset_index()
df_prophet.columns = ["ds", "y"]

model_p = Prophet(yearly_seasonality=True)
model_p.fit(df_prophet)

future = model_p.make_future_dataframe(periods=60)
forecast = model_p.predict(future)
pred_prophet = forecast["yhat"].tail(60)

5. Comparación de Modelos

5.1 Métricas de evaluación

RMSE

√(Σ(y − ŷ)² / n)

Root Mean Squared Error: Penaliza errores grandes. En las mismas unidades que la variable (m³/s).

MAE

Σ|y − ŷ| / n

Mean Absolute Error: Promedio de errores absolutos. Menos sensible a outliers que RMSE.

MAPE

100 · Σ|y − ŷ| / y / n

Mean Absolute Percentage Error: Error porcentual. Permite comparar entre series de diferentes escalas.

6. Pronóstico Futuro (30 días)

Con el mejor modelo identificado (según las métricas del test), se re-entrena con toda la serie (train + test) y se genera un pronóstico para los próximos 30 días más allá del último dato disponible.

# Re-entrenar con toda la serie y pronosticar
model_final = MejorModelo(df_completo, ...)
fit_final = model_final.fit(...)
forecast_30 = fit_final.forecast(steps=30)
💡 ¿Por qué re-entrenar?
Durante la evaluación, separamos datos de test para medir el error. Para el pronóstico final, usamos toda la información disponible para que el modelo tenga el máximo contexto posible. Es la práctica estándar en forecasting.

7. Conclusiones del Proyecto

🎯 Pregunta de investigación

¿Es posible pronosticar el caudal medio diario de la estación Pueblo Nuevo con modelos de series temporales clásicos y modernos?

Respuesta: Sí. SARIMA y Holt-Winters logran capturar la dinámica general del caudal con RMSE≈0.54-0.55 m³/s. Prophet tiene un rendimiento considerablemente peor (RMSE=2.08, MAPE=80.9%). El modelo con menor error (SARIMA) es seleccionado para el pronóstico operativo. El CV≈47.5% genera incertidumbre considerable en predicciones a largo plazo.

Lecciones aprendidas

  1. La calidad de los datos es fundamental: Sin la imputación y el tratamiento de outliers (Semana 2), los modelos de forecasting no convergen o producen resultados poco confiables.
  2. El diagnóstico previo es clave: Los tests de estacionariedad y el análisis ACF/PACF (Semana 3) guían directamente la parametrización de SARIMA.
  3. No existe un modelo universalmente mejor: Cada modelo tiene fortalezas. SARIMA es interpretable y flexible, Holt-Winters es intuitivo, Prophet es automático y robusto.
  4. La evaluación debe ser temporal: El split train/test respeta el orden cronológico — nunca se mezclan datos pasados y futuros.
  5. Siempre medir cuantitativamente: Las métricas (RMSE, MAE, MAPE) proporcionan una comparación objetiva entre modelos.
