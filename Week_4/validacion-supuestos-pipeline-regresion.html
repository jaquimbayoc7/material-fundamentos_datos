Validación de Supuestos y Pipeline Completo

Un buen modelo de regresión no termina con entrenar y predecir. Hay que validar que los supuestos estadísticos se cumplan y construir un pipeline profesional desde la carga de datos hasta la predicción final.

1. Análisis Detallado con Statsmodels

Statsmodels es una librería de Python que proporciona un resumen estadístico completo del modelo de regresión: coeficientes, errores estándar, p-valores, intervalos de confianza, R², y pruebas diagnósticas. Es el complemento perfecto de scikit-learn para interpretar modelos.

¿Por qué usar Statsmodels además de scikit-learn?

scikit-learn

  • Enfocado en predicción
  • API consistente (fit/predict)
  • Ideal para pipelines de ML
  • No proporciona p-valores ni intervalos de confianza

Statsmodels

  • Enfocado en inferencia estadística
  • Proporciona p-valores, IC, pruebas diagnósticas
  • Resumen detallado tipo investigación
  • Ideal para validar y entender el modelo

Modelo OLS con resumen completo

📊 Summary de Statsmodels

import statsmodels.api as sm
import pandas as pd
import numpy as np

url = "https://raw.githubusercontent.com/jaquimbayoc7/material-fundamentos_datos/main/data/water_potability.csv"
df = pd.read_csv(url)

# Imputar nulos
for col in df.columns:
    if df[col].isnull().sum() > 0:
        df[col].fillna(df[col].median(), inplace=True)
df = df.drop_duplicates().reset_index(drop=True)

# Seleccionar features
target = 'Conductivity'
features_ok = ['ph', 'Hardness', 'Solids', 'Chloramines', 'Sulfate',
               'Organic_carbon', 'Trihalomethanes', 'Turbidity']

# Modelo OLS (Ordinary Least Squares)
X_sm = sm.add_constant(df[features_ok])  # Agregar intercepto manualmente
modelo_sm = sm.OLS(df[target], X_sm).fit()

# Resumen completo
print(modelo_sm.summary())

Lectura del Summary

Sección Qué muestra Qué buscar
R-squared Proporción de varianza explicada Cercano a 1 = buen ajuste
Adj. R-squared R² penalizado por número de variables Comparar entre modelos con diferente cantidad de variables
F-statistic Significancia global del modelo Prob(F) < 0.05 → al menos una variable es significativa
coef Valor del coeficiente βᵢ Magnitud y dirección del efecto
P>|t| p-valor individual de cada variable p < 0.05 → variable significativa
[0.025 - 0.975] Intervalo de confianza al 95% Si incluye 0 → variable puede no ser significativa
Durbin-Watson Autocorrelación de residuos Cercano a 2 = no autocorrelación

Identificar variables significativas

📝 Filtrar por p-valor

# Tabla de coeficientes con significancia
coefs = pd.DataFrame({
    'Variable': modelo_sm.params.index[1:],
    'Coeficiente': modelo_sm.params.values[1:],
    'p-valor': modelo_sm.pvalues.values[1:],
    'Significativo': ['✅ Sí' if p < 0.05 else '❌ No'
                      for p in modelo_sm.pvalues.values[1:]]
})
coefs = coefs.sort_values('p-valor')
display(coefs.round(6))

sig_vars = coefs[coefs['p-valor'] < 0.05]['Variable'].tolist()
print(f"Variables significativas (p < 0.05): {sig_vars}")

2. Supuesto 1: Linealidad

Linealidad: Debe existir una relación lineal entre las variables predictoras y la variable objetivo. Si la relación real es curva, el modelo lineal no la capturará adecuadamente.

¿Cómo verificar?

Se grafica Residuos vs Valores Predichos. Si los residuos están dispersos aleatoriamente alrededor de 0, se cumple la linealidad.

✅ Linealidad cumplida

Los puntos se distribuyen aleatoriamente alrededor de la línea roja (y=0). No hay patrón visible.

❌ Linealidad no cumplida

Los puntos forman una curva o patrón (parábola, forma de U). Indica relación no lineal.

📊 Gráfico de residuos vs predichos

import matplotlib.pyplot as plt

residuos = modelo_sm.resid
valores_predichos = modelo_sm.fittedvalues

plt.figure(figsize=(10, 5))

plt.axhline(y=0, color='red', linewidth=1)

plt.ylabel('Residuos', fontsize=12)
plt.title('Residuos vs Valores Predichos (Verificación de Linealidad)')
plt.tight_layout()
plt.show()

print("Si los residuos están dispersos aleatoriamente → linealidad ✓")

3. Supuesto 2: Normalidad de Residuos

Normalidad: Los residuos del modelo deben seguir una distribución normal (campana de Gauss). Este supuesto es necesario para que los p-valores y los intervalos de confianza sean válidos.

Herramientas de diagnóstico

Visual: Histograma

Graficar el histograma de residuos y superponer la curva normal teórica.

Si se ajustan → normalidad.

Visual: Q-Q Plot

Los puntos deben caer sobre la línea recta diagonal.

Desviaciones en las colas indican no normalidad.

Formal: Shapiro-Wilk

H₀: los datos son normales.

Si p ≥ 0.05 → no rechazar H₀ → normalidad.

📊 Histograma + Q-Q Plot

from scipy import stats

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Histograma con curva normal
axes[0].hist(residuos, bins=40, density=True, color='steelblue',
             edgecolor='white', alpha=0.7)
x_norm = np.linspace(residuos.min(), residuos.max(), 100)
axes[0].plot(x_norm, stats.norm.pdf(x_norm, residuos.mean(), residuos.std()),
             'r-', linewidth=2, label='Normal teórica')
axes[0].set_xlabel('Residuos')
axes[0].set_ylabel('Densidad')
axes[0].set_title('Distribución de Residuos')
axes[0].legend()

# Q-Q Plot
stats.probplot(residuos, dist='norm', plot=axes[1])
axes[1].set_title('Q-Q Plot de Residuos')

plt.tight_layout()
plt.show()

# Prueba formal
muestra = np.random.choice(residuos, size=min(5000, len(residuos)), replace=False)
sw_stat, sw_pvalue = stats.shapiro(muestra)
print(f"Shapiro-Wilk: W={sw_stat:.4f}, p={sw_pvalue:.6f}")
print(f"Conclusión: {'✅ Normalidad' if sw_pvalue >= 0.05 else '⚠️ No normalidad'}")

Gráfica Interactiva: Distribución de Residuos

Histograma de los residuos con la curva normal teórica superpuesta:

💡 ¿Qué hacer si no hay normalidad?

  • Con muestras grandes (n > 500), el Teorema Central del Límite suaviza el efecto — el modelo sigue siendo útil
  • Aplicar transformaciones a la variable objetivo (log, raíz cuadrada)
  • Verificar si hay outliers extremos que distorsionan los residuos
  • Considerar modelos no paramétricos o de regresión robusta

4. Supuesto 3: Homoscedasticidad

Homoscedasticidad: La varianza de los residuos debe ser constante a lo largo de todos los valores predichos. Si la dispersión de los residuos cambia (forma de embudo), hay heteroscedasticidad.

Prueba de Breusch-Pagan

Prueba formal que evalúa si la varianza de los residuos depende de las variables independientes.

H₀: Varianza constante (homoscedasticidad)
H₁: Varianza no constante (heteroscedasticidad)
Si p < 0.05 → rechazar H₀ → hay heteroscedasticidad

📊 Scale-Location Plot + Breusch-Pagan

from statsmodels.stats.diagnostic import het_breuschpagan

bp_stat, bp_pvalue, _, _ = het_breuschpagan(residuos, X_sm)

print(f"Breusch-Pagan:")
print(f"  Estadístico LM: {bp_stat:.4f}")
print(f"  p-valor:         {bp_pvalue:.6f}")
print(f"  Conclusión:      {'❌ Heteroscedasticidad' if bp_pvalue < 0.05 else '✅ Homoscedasticidad'}")

# Scale-Location Plot
plt.figure(figsize=(10, 5))
plt.scatter(valores_predichos, np.sqrt(np.abs(residuos)), alpha=0.3, s=10, color='coral')
plt.xlabel('Valores Predichos')
plt.ylabel('√|Residuos|')
plt.title(f'Scale-Location Plot (BP p={bp_pvalue:.4f})')
plt.tight_layout()
plt.show()

Gráfica Interactiva: Homoscedasticidad vs Heteroscedasticidad

Compara cómo se ven los residuos cuando la varianza es constante (izquierda) vs cuando crece con los valores predichos (derecha):

💡 ¿Qué hacer si hay heteroscedasticidad?

  • Transformar la variable objetivo: log(Y) o √Y
  • Usar errores estándar robustos (HC3) en statsmodels
  • Aplicar Weighted Least Squares (WLS)

5. Supuesto 4: Independencia de Errores

Independencia: Los residuos no deben estar correlacionados entre sí. Si un residuo alto precede a otro residuo alto sistemáticamente, hay autocorrelación, lo que invalida los errores estándar y los p-valores.

Prueba de Durbin-Watson

DW ∈ [0, 4]
DW ≈ 2 → No hay autocorrelación ✅
DW < 1.5 → Autocorrelación positiva ⚠️
DW > 2.5 → Autocorrelación negativa ⚠️

📊 Prueba de Durbin-Watson

from statsmodels.stats.stattools import durbin_watson

dw = durbin_watson(residuos)
print(f"Durbin-Watson: {dw:.4f}")

if dw < 1.5:
    print("⚠️ Posible autocorrelación positiva (DW < 1.5)")
elif dw > 2.5:
    print("⚠️ Posible autocorrelación negativa (DW > 2.5)")
else:
    print("✅ No hay autocorrelación significativa (1.5 < DW < 2.5)")

Supuesto 5: No Multicolinealidad

Este supuesto ya se verificó al construir el modelo con el VIF. Variables con VIF < 10 se consideran aceptables.

📐 Resumen de VIF

from statsmodels.stats.outliers_influence import variance_inflation_factor

vif_data = pd.DataFrame()
vif_data['Variable'] = features_ok
vif_data['VIF'] = [variance_inflation_factor(X_sm.values, i+1)
                   for i in range(len(features_ok))]
vif_data = vif_data.sort_values('VIF', ascending=False)
display(vif_data)

print(f"VIF máximo: {vif_data['VIF'].max():.2f}")
print(f"{'✅ No hay multicolinealidad' if vif_data['VIF'].max() < 10 else '⚠️ Multicolinealidad detectada'}")

Resumen de Validación de Supuestos

Supuesto Prueba/Herramienta Criterio Qué hacer si falla
Linealidad Residuos vs Predichos Sin patrón visible Agregar términos cuadráticos o usar modelos no lineales
Normalidad Shapiro-Wilk + Q-Q Plot p ≥ 0.05 Transformar Y o usar modelos robustos
Homoscedasticidad Breusch-Pagan p ≥ 0.05 Transformar Y, WLS o errores robustos
Independencia Durbin-Watson 1.5 ≤ DW ≤ 2.5 Incluir rezagos o usar modelos de series temporales
No multicolinealidad VIF VIF < 10 Eliminar o combinar variables correlacionadas

6. Pipeline Completo de Regresión

Un pipeline profesional integra todo lo aprendido en las 4 semanas en un flujo reproducible. Aquí mostramos los 8 pasos desde la carga hasta la predicción final.

📥 Carga de Datos

import pandas as pd
import numpy as np

url = "https://raw.githubusercontent.com/jaquimbayoc7/material-fundamentos_datos/main/data/water_potability.csv"
df_pipe = pd.read_csv(url)
print(f"Datos cargados: {df_pipe.shape}")

🧹 Limpieza de Datos

# Imputar nulos con mediana
for col in df_pipe.columns:
    nulos = df_pipe[col].isnull().sum()
    if nulos > 0:
        df_pipe[col].fillna(df_pipe[col].median(), inplace=True)
        print(f"  {col}: {nulos} nulos imputados")

# Eliminar duplicados
dupl = df_pipe.duplicated().sum()
df_pipe = df_pipe.drop_duplicates().reset_index(drop=True)
print(f"  Duplicados eliminados: {dupl}")

🔧 Ingeniería de Características

# Crear nuevas variables que puedan aportar información
df_pipe['ratio_solidos_cond'] = df_pipe['Solids'] / df_pipe['Conductivity']
df_pipe['ph_neutral'] = ((df_pipe['ph'] >= 6.5) & (df_pipe['ph'] <= 8.5)).astype(int)
df_pipe['log_solids'] = np.log1p(df_pipe['Solids'])
print("3 nuevas características creadas")

🎯 Selección de Variables

target_pipe = 'Conductivity'
features_pipe = ['ph', 'Hardness', 'Solids', 'Chloramines', 'Sulfate',
                 'Organic_carbon', 'Trihalomethanes', 'Turbidity',
                 'ratio_solidos_cond', 'log_solids']

print(f"Target: {target_pipe}")
print(f"Features: {len(features_pipe)} variables")

✂️ División Train/Test

from sklearn.model_selection import train_test_split

X_pipe = df_pipe[features_pipe].values
y_pipe = df_pipe[target_pipe].values

X_train_p, X_test_p, y_train_p, y_test_p = train_test_split(
    X_pipe, y_pipe, test_size=0.2, random_state=42
)
print(f"Train: {X_train_p.shape[0]} | Test: {X_test_p.shape[0]}")

📏 Estandarización

from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train_p)  # fit + transform en train
X_test_scaled = scaler.transform(X_test_p)         # solo transform en test
print("Features estandarizadas (media=0, std=1)")

⚠️ Importante

El fit del scaler se hace solo en entrenamiento. El test se transforma con los parámetros del train para evitar data leakage (filtración de información del futuro al modelo).

🏋️ Entrenamiento

from sklearn.linear_model import LinearRegression

modelo_final = LinearRegression()
modelo_final.fit(X_train_scaled, y_train_p)
print("Modelo entrenado")

📊 Evaluación Final

from sklearn.metrics import r2_score, mean_squared_error, mean_absolute_error

y_pred_final = modelo_final.predict(X_test_scaled)

r2_final = r2_score(y_test_p, y_pred_final)
rmse_final = np.sqrt(mean_squared_error(y_test_p, y_pred_final))
mae_final = mean_absolute_error(y_test_p, y_pred_final)

# R² ajustado
n_f = X_test_scaled.shape[0]
p_f = X_test_scaled.shape[1]
r2_adj_final = 1 - (1 - r2_final) * (n_f - 1) / (n_f - p_f - 1)

print(f"R²:          {r2_final:.4f}")
print(f"R² Ajustado: {r2_adj_final:.4f}")
print(f"RMSE:        {rmse_final:.4f}")
print(f"MAE:         {mae_final:.4f}")

Visualización del Pipeline

🎨 Gráfico 4-en-1 del modelo final

fig, axes = plt.subplots(2, 2, figsize=(14, 12))

# 1. Real vs Predicho
residuos_f = y_test_p - y_pred_final
axes[0,0].scatter(y_test_p, y_pred_final, alpha=0.3, s=15, color='steelblue')
lim = [min(y_test_p.min(), y_pred_final.min()), max(y_test_p.max(), y_pred_final.max())]
axes[0,0].plot(lim, lim, 'r--', linewidth=1)
axes[0,0].set_title(f'Real vs Predicho (R²={r2_final:.4f})')

# 2. Distribución de residuos
axes[0,1].hist(residuos_f, bins=35, color='coral', edgecolor='white', density=True)
axes[0,1].set_title('Distribución de Residuos')

# 3. Residuos vs Predichos
axes[1,0].scatter(y_pred_final, residuos_f, alpha=0.3, s=15, color='mediumseagreen')
axes[1,0].axhline(0, color='red', linewidth=1)
axes[1,0].set_title('Residuos vs Predichos')

# 4. Importancia de variables
importancia = pd.Series(np.abs(modelo_final.coef_), index=features_pipe).sort_values()
importancia.plot(kind='barh', color='steelblue', ax=axes[1,1])
axes[1,1].set_title('Importancia de Variables')

plt.suptitle('Pipeline Completo — Regresión Lineal Múltiple', fontsize=16, fontweight='bold')
plt.tight_layout()
plt.show()

Predicción con datos nuevos

🔮 Ejemplo de predicción

nueva_muestra = {
    'ph': 7.0, 'Hardness': 200.0, 'Solids': 20000.0,
    'Chloramines': 7.0, 'Sulfate': 330.0, 'Organic_carbon': 14.0,
    'Trihalomethanes': 65.0, 'Turbidity': 4.0,
    'ratio_solidos_cond': 20000.0 / 400.0,
    'log_solids': np.log1p(20000.0)
}

X_nuevo = pd.DataFrame([nueva_muestra])[features_pipe].values

8. Resumen del Curso Completo

A lo largo de las 4 semanas, construimos un pipeline completo de análisis de datos ambientales:

Semana 1 — Carga y Visualización

Carga desde múltiples fuentes. Exploración con pandas. Visualización con Matplotlib, Seaborn y Plotly Express.

Semana 2 — Calidad y Transformación

Diagnóstico de calidad. Limpieza de nulos, duplicados y outliers. Normalización, encoding y feature engineering.

Semana 3 — Análisis Estadístico

Correlaciones (Pearson, Spearman, Kendall). Supuestos estadísticos. Pruebas de hipótesis (t-test, ANOVA, Shapiro-Wilk).

Semana 4 — Regresión Lineal

Regresión simple y múltiple. Métricas de evaluación. Validación de 5 supuestos. Pipeline completo de predicción.

🎯 Lecciones Aprendidas

  1. La preparación de datos (limpieza, transformación) es fundamental para obtener buenos modelos
  2. Siempre debemos validar los supuestos antes de interpretar los resultados
  3. Un R² bajo no significa un modelo inútil — puede indicar que las variables disponibles no capturan toda la variabilidad del fenómeno
  4. La ingeniería de características puede mejorar significativamente el rendimiento del modelo
  5. Trabajar con datos ambientales reales implica lidiar con ruido, valores atípicos y relaciones complejas
